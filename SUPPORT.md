# Support

IRIN is local-first software for one operator. Community support responses are best-effort (no paid SLA).

For usage questions, setup problems, and reproducible bugs, search the documentation and existing issues, then open the appropriate GitHub issue form. Include the affected version or commit, your operating system, reproduction steps, and sanitized logs where useful.

Do not include credentials, signing keys, provider tokens, private prompts, session content, or other sensitive local data in an issue.

Report vulnerabilities privately through [GitHub Security Advisories](https://github.com/irinityhq/irin/security/advisories/new) or email `soc@irinity.com`. Do not open a public issue for a vulnerability. See [SECURITY.md](SECURITY.md) for the supported version and security boundary.
